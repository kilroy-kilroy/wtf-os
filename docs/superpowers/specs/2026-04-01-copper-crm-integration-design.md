# Copper CRM Integration — Design Spec

**Date:** 2026-04-01
**Status:** Draft

## Overview

Integrate Copper CRM across all lead capture, report generation, and subscription flows. New leads get a Copper person + opportunity. Report generation adds notes to existing contacts. Stripe purchases close deals as won. All Copper calls are fire-and-forget (non-blocking).

## Goals

1. Every email capture creates a Copper person and pipeline opportunity
2. Every report generated by an existing user adds a note to their Copper contact
3. Stripe subscription purchases close the matching opportunity as won
4. One opportunity per product per contact (not per report)
5. Zero impact on user-facing latency (all fire-and-forget)

## Non-Goals

- Syncing data FROM Copper back into the app
- Bulk importing existing users into Copper (can be done separately)
- Copper webhooks (one-directional: app → Copper)
- UI for viewing Copper data in the admin panel

## 1. Copper Utility Module

**File:** `apps/web/lib/copper.ts`

### Configuration

- `COPPER_API_KEY` — env var (already added to Vercel)
- `COPPER_API_EMAIL` — env var (already added to Vercel)
- Base URL: `https://api.copper.com/developer_api/v1`
- Auth headers on every request:
  - `X-PW-AccessToken: {COPPER_API_KEY}`
  - `X-PW-UserEmail: {COPPER_API_EMAIL}`
  - `X-PW-Application: developer_api`
  - `Content-Type: application/json`

### Pipeline Constants

```typescript
const PIPELINE_ID = 1044966;

const STAGES = {
  LEAD: 5005078,
  DISCOVERY_CALL: 5005079,
  QUALIFIED: 5005080,
  ALIGNMENT: 5005081,
  VERBAL: 5005082,
  CONTRACT: 5005083,
  CLOSED_WON: 5005084,
  CLOSED_LOST: 5005085,
};

const PRO_ACV = 58800; // $588 in cents (Copper uses cents for monetary_value)
```

### Functions

**`copperFindOrCreatePerson(email, name?, companyName?, websiteUrl?)`**
- Calls `POST /people/search` with `{emails: [email]}`
- If found, returns the first match's ID
- If not found, calls `POST /people` to create with name, email, company_name
- Returns `{ id: string } | null`

**`copperCreateOpportunity(personId, productName, monetaryValue, stageId)`**
- Calls `POST /opportunities` with:
  - `name`: `"{productName} — {personEmail}"`
  - `pipeline_id`: PIPELINE_ID
  - `pipeline_stage_id`: stageId
  - `monetary_value`: monetaryValue (in cents)
  - `primary_contact_id`: personId
- Returns `{ id: string } | null`

**`copperFindOpportunity(personId, productName)`**
- Calls `POST /opportunities/search` with `{primary_contact_id: personId}`
- Filters results client-side: finds one where `name` starts with `productName`
- Returns `{ id: string, pipeline_stage_id: number } | null`

**`copperUpdateOpportunity(opportunityId, updates)`**
- Calls `PUT /opportunities/{id}` with stage and/or value updates
- Returns `{ success: boolean }`

**`copperAddNote(personId, text)`**
- Calls `POST /activities` with:
  - `type`: note activity type (need to look up ID via List Activity Types, or use `{id: 0, category: "user"}`)
  - `details`: text content
  - `parent`: `{type: "person", id: personId}`
- Returns `{ success: boolean }`

**`copperFindOrCreateOpportunity(personId, personEmail, productName, monetaryValue, stageId)`**
- Convenience function: finds existing opportunity for this product, or creates one
- Used at lead capture time to avoid duplicate opportunities
- Returns the opportunity ID

### Error Handling

All functions:
- Wrapped in try/catch
- Log errors to `console.error('[Copper] ...')`
- Return null/false on failure
- Never throw (fire-and-forget callers shouldn't need to handle errors)

## 2. Integration Points

### New Lead Capture (create person + opportunity)

**GrowthOS Assessment** — `apps/web/app/api/growthos/route.ts`
After successful assessment (after Loops/Beehiiv calls, fire-and-forget):
```
Find/create person: email=intakeData.email, name=intakeData.founderName, company=intakeData.agencyName, website=intakeData.website
Create opportunity: "WTF Assessment", value=$0, stage=Lead
Add note: "Completed WTF Assessment — Score: {scores.overall}/5. View: {appUrl}/growthos/results/{assessmentId}"
```

**Visibility Lab Free** — `apps/web/app/api/visibility-lab/analyze/route.ts`
After successful report generation:
```
Find/create person: email=input.userEmail, name=input.userName, company=input.brandName
Find/create opportunity: "Visibility Lab Pro", value=$588 ACV, stage=Lead
Add note: "Ran Visibility Lab Free — Score: {visibilityScore}/100. View: {appUrl}/visibility-lab/report/{reportId}"
```

**Visibility Lab Pro** — `apps/web/app/api/visibility-lab-pro/analyze/route.ts`
After successful report generation:
```
Find/create person: email=input.userEmail, name=input.userName, company=input.brandName
Find/create opportunity: "Visibility Lab Pro", value=$588 ACV, stage=Discovery Call (higher intent)
Add note: "Ran Visibility Lab Pro — Score: {compositeScore}/100. View: {appUrl}/visibility-lab/report/{reportId}"
```

**Call Lab Instant** — `apps/web/app/api/call-lab-instant/capture-lead/route.ts`
After email capture:
```
Find/create person: email=email, name=firstName
Find/create opportunity: "Call Lab Pro", value=$588 ACV, stage=Lead
Add note: "Call Lab Instant report — Score: {score}/10. View: {appUrl}/call-lab/report/{reportId}"
```

**Discovery Lab** — `apps/web/app/api/analyze/discovery/route.ts`
After report generation:
```
Find/create person: email=requestor_email, name=requestor_name, company=requestor_company
Find/create opportunity: "Discovery Lab Pro", value=$588 ACV, stage=Lead
Add note: "Ran Discovery Lab {version} — Target: {target_company}. View: {appUrl}/discovery-lab/report/{reportId}"
```

### Existing User Report (update contact + add note)

**Call Lab Pro/Lite** — `apps/web/app/api/analyze/call/route.ts`
In the `sendReportEmail` helper function, after Loops event:
```
Find person by email
If found:
  Add note: "Ran Call Lab {tier} report — Score: {score}. View: {appUrl}/call-lab/report/{reportId}"
  Find/create opportunity: "Call Lab Pro", value=$588 ACV, stage=Lead (only if no existing opp)
```

### Subscription Purchase (close deal)

**Stripe Checkout** — `apps/web/app/api/webhooks/stripe/route.ts`
In the `checkout.session.completed` handler, after subscription record creation:
```
Find person by email (session.customer_email)
Map product metadata to opportunity name:
  'call-lab-pro' → "Call Lab Pro"
  'discovery-lab-pro' → "Discovery Lab Pro"
  'visibility-lab-pro' → "Visibility Lab Pro"
  'growth-bundle' / 'growthos-bundle' → "Growth Bundle"
Find opportunity matching product name
If found: update to Closed-Won, set value to ACV
If not found: create as Closed-Won with ACV
Add note: "Purchased {product} ({planType} plan) — Subscription: {subscriptionId}"
```

ACV values by product:
- Call Lab Pro: $588 (58800 cents)
- Discovery Lab Pro: $588 (58800 cents)
- Visibility Lab Pro: $588 (58800 cents)
- Growth Bundle: $1764 (176400 cents, 3 × $588)

## 4. Calling Pattern

Every integration point follows the same pattern — a fire-and-forget async block added after the existing Loops/Beehiiv calls:

```typescript
// Copper CRM (fire-and-forget)
copperSyncLead({
  email,
  name,
  companyName,
  productName: 'Visibility Lab Pro',
  opportunityValue: PRO_ACV,
  stage: STAGES.LEAD,
  note: `Ran Visibility Lab Free — Score: ${score}/100. View: ${appUrl}/visibility-lab/report/${reportId}`,
}).catch(err => console.error('[Copper] sync failed:', err));
```

To reduce boilerplate, `copper.ts` will expose a high-level `copperSyncLead()` function that orchestrates find/create person → find/create opportunity → add note in a single call. This is what each route calls.

Similarly, `copperCloseDeal()` for the Stripe webhook: find person → find/update opportunity → add note.

## 5. Edge Cases

- **Duplicate emails**: `copperFindOrCreatePerson` searches first, only creates if not found. Safe for repeat submissions.
- **Duplicate opportunities**: `copperFindOrCreateOpportunity` searches by person + product name prefix. Only creates if none exists.
- **Copper API down**: All calls are fire-and-forget with error logging. User flow unaffected.
- **Missing env vars**: `copper.ts` checks for `COPPER_API_KEY` and `COPPER_API_EMAIL` at the top of each function. If missing, logs warning and returns early (same pattern as Loops).
- **User runs free then pro**: Free creates opportunity at "Lead" stage. Pro report finds existing opportunity — if still at "Lead", bumps to "Discovery Call". If already advanced, leaves it.
- **User subscribes to product with no existing opportunity**: Stripe handler creates a new opportunity directly at "Closed - Won".
- **Monetary value**: Copper uses cents (integer). $588 = 58800. $0 = 0.
